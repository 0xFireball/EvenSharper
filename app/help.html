How it works

EvenSharper runs on FireHTTPServer, also created by 0xFireball. Every time you press the Compile button, your code is sent to the server, which has a powerful built-in engine for scripting ExaScript on the .NET Framework. FireHTTPServer and ExaScript run on Windows out-of-the-box, and on OS X and Linux with the Mono Framework.

Help/Tips

  • Your code is saved in HTML5 local storage, so it will not be lost every time you refresh. However, if you clear stored site data, your code will be lost. For this reason, we recommend storing your raw code in text format locally, and pasting into the text editor only for compilation.
  • More stuff is hopefully coming soon!
  • The type of assembly (DLL or EXE) is inferred based on the file extension!

Technical Info

Referenced Assemblies

Only a subset of all of the .NET Framework assemblies are referenced. By default, you get all of the assemblies from the following list. The assemblies are sufficient for self-contained command-line executables and Windows Forms GUI applications.

  • mscorlib.dll
  • System.dll
  • System.Core.dll
  • System.Windows.Forms.dll
